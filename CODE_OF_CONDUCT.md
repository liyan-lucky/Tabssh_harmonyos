# Code of Conduct

## Our Pledge

We want TabSSH / OpenTabSsh to be a respectful, practical, and safe open source project. Contributors, users, and maintainers are expected to communicate professionally and focus on improving the project.

## Expected Behavior

Examples of behavior that helps the project:

- Be respectful and constructive.
- Discuss ideas, code, and technical decisions without personal attacks.
- Assume good intent when possible.
- Keep reports and pull requests focused and reproducible.
- Avoid sharing private credentials, private infrastructure details, or sensitive logs.
- Respect third-party licenses, trademarks, and platform terms.

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, insults, threats, or discriminatory language.
- Publishing another person's private information without permission.
- Sharing real credentials, private keys, tokens, or private service details in public issues or pull requests.
- Encouraging illegal access, unauthorized intrusion, credential theft, or abuse of SSH functionality.
- Removing or obscuring third-party copyright notices and license terms.
- Misrepresenting this project as an official Huawei, HarmonyOS, OpenHarmony, OpenSSH, SSH, or libssh2 product.

## Enforcement

Maintainers may remove comments, close issues, reject pull requests, or block participants who violate this Code of Conduct.

Maintainers should apply these rules fairly and proportionally. Serious security, privacy, legal, or abuse-related incidents may be handled immediately.

## Reporting

If you see behavior that violates this Code of Conduct, contact the maintainers through an appropriate private channel when possible. Do not include private keys, passwords, tokens, or sensitive logs in public reports.

## Scope

This Code of Conduct applies to project spaces, including GitHub issues, pull requests, discussions, documentation, release notes, and community channels controlled by the project maintainers.
